Pipeline flow
1
HubSpot Breeze
Visitor identification
Identifies companies visiting sociallinks.io via Buyer Intent. Auto-add to CRM is ON via "Weekly digest" saved view. ~100 companies/week. Data lives in Marketing → Buyer Intent → Visitors, not standard CRM fields.
2
Make.com — Scenario 1
HubSpot → Clay webhook
Watches HubSpot for new Companies (Created only). Every 15 min sends payload to Clay webhook. Payload: domain + Country/Region (added June 19, 2026). Module: Watch CRM Objects → Clay "Create a record in table". Scenario URL: eu1.make.com/1903842/scenarios/6146312
3
Clay — Companies table
Company enrichment
Receives domain via webhook. Enriches via People Data Labs (company name, domain). 170 rows, 8 columns. HTTP API column is legacy — kept but replaced by Contacts table flow.
4
Clay — Contacts table
ICP contact finding + email waterfall
Source: "Security and Risk Leadership" filter — Find people with filters, scoped to Companies table via Website column. Job title: "is similar to" + 183 ICP titles. Limit: 5 per company. Email waterfall: Findymail → Prospeo → Hunter → Dropcontact → Datagma → Wiza. HTTP API column POSTs to Make.com. Run condition: !!{Work Email}. Currently: 109 rows, 27 columns.
5
Make.com — Scenario 2
Clay → HubSpot contacts
Receives Clay HTTP API POST. Sleep 1s → Create or Update Contact (dedup by email). Process data in order: ON. Scenario URL: eu1.make.com/1903842/scenarios/6158364
6
HubSpot
Verified ICP contacts in CRM
Contacts appear under All Contacts sorted by Create Date. SDR team works from here. Pipeline contacts identified by: company_website property has a value.

Known constraints
⚠
Clay Explorer — no native HubSpot sync. Requires Growth ($495/mo). Using Make.com HTTP webhook bridge instead. If upgrading: replace Scenario 2 with Clay's native HubSpot export.
⚠
Arthur has limited HubSpot permissions. Admin required for Make.com install and Backfill. Cannot create companies.
⚠
Dropcontact currently skipped. Needs full name + company name. Make.com Scenario 1 only sends domain. To unlock: add company_name to payload.
ℹ
Make.com ops usage: monitor monthly consumption. Each contact = 2 ops (sleep + HubSpot module).
ℹ
Pipeline contacts filter in HubSpot: contacts where company_website has a value = came from ABM pipeline. Use this filter for the Cowork digest.
⚠
ICP filter noise. Some non-ICP contacts slip through (airline captains match "Captain", researchers match "Investigator"). Exclusion rules to be added later. Current workaround: limit 5 per company reduces damage.
