Pipeline Flow
1
HubSpot Breeze
Visitor Identification
Identifies visiting companies via Buyer Intent. Auto-add to CRM is ON via "Weekly digest" saved view.
2
Make.com — Scenario 1
HubSpot → Clay
Watches for new Companies in HubSpot (Created only). Sends domain to Clay webhook every 15 minutes.
3
Clay — ABM Pipeline table
Company Enrichment
Receives domain via webhook. Enriches company data via People Data Labs (name, domain, size).
4
Clay — Contacts table
ICP Contact Finding
Finds contacts at each company matching ICP job titles (security, intelligence, compliance, LEA, cyber). Limit: 3 per company. Email waterfall: Findymail → Prospeo → Hunter → Dropcontact → Datagma → Wiza. ZeroBounce validates final email.
5
Make.com — Scenario 2
Clay → HubSpot
Clay HTTP API column POSTs verified contacts to Make.com webhook. Only runs when Work Email is not empty. Make.com creates/updates contact in HubSpot with: email, first name, last name, job title, LinkedIn URL, company domain.
6
HubSpot
Verified ICP Contacts Live in CRM
Contacts appear under All Contacts sorted by Create Date. SDR team picks up from here.

Clay Tables
ABM Pipeline
One row per company. Source: Webhook (domain). Enrichment: PDL company name + domain. Columns: domain, company name, Hunter emails, ZeroBounce status, HTTP API (sends to Make.com — legacy, now replaced by Contacts table).
Contacts
One row per person. Source: Find Contacts at Company (Clay native, ICP job titles filter). Email waterfall auto-built by Clay. HTTP API column sends verified contacts to Make.com Scenario 2. Run condition: !!{Work Email}
Email Waterfall — Contacts Table

Clay built this waterfall automatically when "Work Email" enrichment was selected. Each step only runs if the previous step found no email — saving credits. The final Work Email column holds the best result.

# Tool Cost Logic
1 Findymail ~2.9 credits Always runs first
2 Prospeo ~2.9 credits Runs if Findymail returns no email
3 Hunter ~2.9 credits Runs if Prospeo returns no email
4 Dropcontact ~2.9 credits Runs if Hunter returns no email. Good European coverage.
5 Datagma ~2.9 credits Runs if Dropcontact returns no email
6 Wiza ~2.9 credits Runs if Datagma returns no email
✓ Work Email (output column) — Best email found across all steps. This is what gets sent to HubSpot.

Important Constraints
  • Clay Explorer plan — no native HubSpot CRM sync (requires Growth $495/mo). Using Make.com HTTP webhook bridge instead.
  • HubSpot — Arthur has limited permissions (cannot create companies). Admin required for Make.com install and Backfill.
  • Apollo free plan exhausted. Not currently in use — replaced by Clay's native "Find Contacts at Company" source.
  • Dropcontact skipped — requires full name + domain. Webhook only sends domain. Can be added if Make.com payload is expanded.
  • Breeze data lives in Marketing → Buyer Intent → Visitors, not in standard CRM company fields.
  • ICP filtering: Clay Contacts table filters by job title at source. HTTP API column has run condition !!{Work Email} to skip empty rows.
